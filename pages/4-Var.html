Entendiendo "var" en JavaScript

La palabra clave var era la principal forma de declarar variables en JavaScript. Aún es importante entender cómo funciona, ya que tiene un comportamiento diferente al de let y const.

1. Declarar y Reasignar Variables con var

Puedes declarar una variable con var y reasignarla fácilmente. 

var nombre = "Carlos";
console.log(nombre); // "Carlos"

nombre = "Ana";
console.log(nombre); // "Ana"
2. Alcance de var

Las variables declaradas con var tienen alcance de función. Si se declaran fuera de una función, son globales. 

var globalVar = "Soy global";

function mostrarVariable() {
  console.log(globalVar); // "Soy global"
}
mostrarVariable();
3. Hoisting (Elevación)

JavaScript eleva las declaraciones con var al inicio de su ámbito, lo que permite usar variables antes de su declaración. 

console.log(edad); // undefined
var edad = 25;
console.log(edad); // 25
4. Problemas con var

El alcance global y el hoisting pueden causar errores difíciles de detectar. Por ejemplo, variables dentro de bucles son accesibles fuera del mismo. 

for (var i = 0; i < 3; i++) {
  console.log(i); // 0, 1, 2
}
console.log(i); // 3

Aunque var aún se usa, es recomendable preferir let o const para un comportamiento más predecible en el alcance de las variables.
