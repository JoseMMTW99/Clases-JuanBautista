Entendiendo "let" en JavaScript

La palabra clave let es una forma más moderna de declarar variables en JavaScript. Introducida en ES6, let tiene un comportamiento diferente a var, solucionando algunos de sus problemas.

1. Declarar y Reasignar Variables con let

Al igual que var, puedes declarar y reasignar variables usando let. 

let nombre = "María";
console.log(nombre); // "María"

nombre = "Pedro";
console.log(nombre); // "Pedro"
2. Alcance de let

A diferencia de var, let tiene alcance de bloque, es decir, solo existe dentro de las llaves donde es declarado. 

if (true) {
  let mensaje = "Dentro del bloque";
  console.log(mensaje); // "Dentro del bloque"
}

// console.log(mensaje); // Error: mensaje no está definido
3. Hoisting (Elevación)

Aunque let también es "elevado", no puedes usar la variable antes de su declaración. Esto es diferente de var. 

console.log(saludo); // Error: no se puede acceder a 'saludo'
let saludo = "Hola";
4. Uso de let en bucles

Las variables declaradas con let en bucles tienen un nuevo valor en cada iteración, lo que evita errores comunes con var. 

for (let i = 0; i < 3; i++) {
  console.log(i); // 0, 1, 2
}

// console.log(i); // Error: i no está definido fuera del bucle

En resumen, let es más seguro y predecible que var. Es preferible usar let en lugar de var para evitar problemas de alcance y hoisting.
