Entendiendo "const" en JavaScript

La palabra clave const es otra forma de declarar variables en JavaScript, introducida junto con let en ES6. Las variables declaradas con const no pueden ser reasignadas, aunque las propiedades internas de los objetos o arrays declarados con const sí pueden cambiar.

1. Declarar una variable con const

Al declarar una variable con const, es obligatorio asignarle un valor inmediatamente. Después de su asignación, no puede ser reasignada. 

const nombre = "Juan";
console.log(nombre); // "Juan"

// nombre = "Pedro"; // Error: no se puede reasignar una variable const
2. Alcance de Bloque con const

Al igual que let, const tiene un alcance de bloque, lo que significa que solo es accesible dentro del bloque en el que fue declarada. 

if (true) {
  const mensaje = "Este es un mensaje";
  console.log(mensaje); // "Este es un mensaje"
}

// console.log(mensaje); // Error: mensaje no está definido fuera del bloque
3. Constantes con Objetos y Arrays

Aunque no puedes reasignar una variable const, los objetos y arrays declarados con const sí pueden modificarse internamente. 

const persona = {
  nombre: "Carlos",
  edad: 30
};

persona.nombre = "Ana"; // Cambia el nombre
console.log(persona.nombre); // "Ana"
4. Hoisting (Elevación)

Al igual que let, const es elevado, pero no puedes usar la variable antes de su declaración. 

console.log(ciudad); // Error: no se puede acceder a 'ciudad'
const ciudad = "Madrid";

Usar const garantiza que una variable no será reasignada, lo que ayuda a evitar errores de lógica. Utilízalo siempre que sepas que el valor de una variable no cambiará.
